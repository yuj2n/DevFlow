# [AI 협업 기록] 12주차: Swagger API 명세 동적 파싱 엔진 및 타입 안정성 구축

본 문서는 **DevFlow** 프로젝트 구현 과정 중 12주차에 진행된 외부 OpenAPI Specification(Swagger) 데이터 연동, 에디터 파싱 레이어 개발, 그리고 이 과정에서 발생한 정적 타입 에러 및 패키지 충돌 결함 해결에 관한 AI(Gemini)와의 협업 내역을 투명하게 기록한 작업 일지입니다.

## 2. 상세 협업 로그

### 🛠️ [LOG-1] Swagger JSON 스펙 기반 데이터 구조 분석 및 파싱 전략 수립

- **프롬프트 (질문):** "원격 Swagger API의 JSON URL을 받아왔어. 이 내부의 엔드포인트 객체 배열이랑 파라미터 구조를 파싱해서 Tiptap 에디터 본문에 맞는 HTML 테이블 구조로 한 번에 치환해서 이식하는 효율적인 가공 함수를 어떻게 짜야 할까?"
- **AI 제공 해결책:**
- Swagger/OpenAPI Spec 규격의 핵심 축인 `paths`, `parameters`, `responses` 객체 노드를 재귀적으로 탐색하고 데이터를 정제하는 파서 핸들러 구조 제시.
- 리액트 기반 WYSIWYG 에디터인 `Tiptap`이 파싱 런타임 오류 없이 안전하게 노드를 읽고 렌더링할 수 있도록 표준 시맨틱 HTML 태그 기반의 데이터 바인딩 포맷팅 가이드라인 제공.

- **직접 수행한 작업:**
- 외부 Swagger API 명세 스트링을 주입받아 자바스크립트 오브젝트로 변환 후 엔드포인트 세부 스펙을 동적으로 순회하는 가공 파이프라인 함수 구현.
- 수동으로 엔드포인트를 하나씩 타이핑하며 제작해야 했던 문서화 워크플로우를 **URL 입력 및 파싱 단 1초 만에 템플릿화**하는 자동화 코어 엔진 연동 성공.

### 🎨 [LOG-2] Tiptap 스키마 호환형 API 명세 레이아웃 가공

- **프롬프트 (질문):** "파싱한 데이터를 에디터에 그냥 집어넣으니까 테이블 선이 깨지고 코드 블록 포맷이 망가져. Tiptap 에디터 규칙을 깨뜨리지 않으면서 깃허브 마크다운 뷰어처럼 예쁘게 나오게 하는 HTML 태그 조합이 뭐야?"
- **AI 제공 해결책:**
- Tiptap의 텍스트 확장 스키마가 허용하는 인라인 스타일 속성(`border-collapse`, `padding`, `background-color`)을 결합한 안정적인 테이블 태그 구성안 제시.
- API 응답 예시(Response JSON) 시각화를 위해 Tiptap이 감지하는 최상위 노드 쌍 태그 포맷인 `<pre><code>...</code></pre>` 컨테이너 가두기 규칙 설명.

- **직접 수행한 작업:**
- 파서가 뱉어내는 텍스트 데이터 주위에 `<thead>`, `<tbody>`, `<tr>`, `<td>` 등 구조화된 웹 표준 레이아웃 태그를 동적으로 감싸주는 문자열 맵핑 로직 완성.
- API의 핵심 정보인 [HTTP Method, Endpoint URL, Request Parameters, Response JSON 데이터]가 에디터 화면에 진입하는 순간 정형화된 그리드로 챡 정렬되어 출력되도록 UI 완성도를 고도화함.

### ⚠️ [LOG-3] 외부 변환 모듈(turndown) 설치 실패 및 패키지 충돌 무력화

- **프롬프트 (질문):** "에디터 HTML을 마크다운으로 추출하기 위해 `turndown` 라이브러리를 설치하려는데, 터미널에서 `Cannot read properties of null (reading 'matches')` 에러가 나면서 빌드가 완전히 멈춰버려. `--legacy-peer-deps` 옵션을 써도 안 되는데 방법이 없어?"
- **AI 제공 해결책:**
- 해당 에러는 특정 라이브러리의 결함이 아닌, 현재 프로젝트 환경 내 패키지들 간의 상호 의존성 트리(Dependency Tree) 계산이 무너진 고질적인 npm 버전 버그임을 진단.
- 마감 임박 상황에서 프로젝트 전반의 버전을 강제 수정하는 리스크를 안는 대신, 외부 모듈 의존성을 0%로 낮추고 런타임 환경에서 가볍게 동작하는 **순수 정규 표현식 문자열 치환(Pure Regex Parsing) 정공법**으로 선회하는 아키텍처 제안.

- **직접 수행한 작업:**
- 설치가 불가능했던 외부 모듈 도입을 취소하여 프로젝트 패키지 오염 및 빌드 차단 결함을 원천 방어함.
- 자바스크립트 내장 정규식 엔진을 활용해 Tiptap이 뱉어내는 특정 HTML 구조를 자체 컴파일하여 마크다운 파일로 세탁하는 고신뢰성 인하우스(In-house) 파서 로직을 선제 구축함.

### 🛡️ [LOG-4] TypeScript 리터럴 타입 할당 오류 대응 (Type Mismatch 해결)

- **프롬프트 (질문):** "Swagger 컴포넌트(SwaggerImport.tsx)에서 외부 URL로 JSON을 받아오거나 파일 업로드를 읽어올 때, 타입스크립트가 데이터를 자꾸 any 상태로 유추해서 타입 세이프하지 못하다는 경고가 발생해. 이걸 정석대로 방어하는 코딩이 뭐야?"
- **AI 제공 해결책:**
- HTTP 통신 라이브러리인 axios 응답 객체와 자바스크립트 내장 JSON.parse() 엔진은 기본적으로 런타임 변수 유형을 추론할 수 없어 컴파일 타임에 any 타입을 반환하는 특징 설명.
- 외부 인프라 레이어에서 비정형 데이터가 유입되는 즉시 프로젝트에 정의된 정적 데이터 모델 인터페이스인 SwaggerData 리터럴 타입을 강제로 주입 및 단언(Type Assertion)해 주는 타입 게이트웨이 설계 유도.

- **직접 수행한 작업:**
- 외부 비동기 네트워킹 호출 인터페이스를 axios.get<SwaggerData>(...) 제네릭 포맷으로 정밀 수정하여 수신 데이터의 any 오염을 초동 조치함.
- 로컬 드래그 앤 드롭으로 업로드된 JSON 파일을 파일 리더기로 파싱하는 구역(FileReader.onload)에서도 JSON.parse(result) as SwaggerData 형태로 명시적 타입 단언(Type Assertion)을 적용해 컴파일러가 객체 내부 구조를 IntelliSense로 자동 완성 및 검증할 수 있는 무결성 런타임 환경 완성.
